# Запуск реранкера на Kaggle

Скрипт `capstone_kaggle.py` проверяет двухэтапную схему:

1. EASE выбирает top-100 кандидатов.
2. LightGBM меняет их порядок по скорам моделей и признакам товаров.

Реранкер обучается на validation-пользователях и оценивается на test.

## Подготовка данных

Загрузи содержимое `artifacts/capstone_kaggle/` как Kaggle Dataset. В нём должны быть:

- `data/` с train, holdouts, каталогом и признаками товаров;
- `ease/` с сохранённой матрицей EASE;
- `src/` с кодом моделей и реранкера.

Название датасета может быть любым: скрипт ищет `src/models/ease.py` внутри
`/kaggle/input/` и сам определяет корневую папку.

## Настройка ноутбука

- добавь созданный Dataset через **Add Input**;
- выбери GPU Accelerator;
- включи Internet для загрузки `all-MiniLM-L6-v2`;
- перенеси секции из `scripts/capstone_kaggle.py` в отдельные ячейки и запусти по порядку.

Размер выборки задаётся переменными:

```python
N_TRAIN_USERS = 150_000
N_EVAL_USERS = 150_000
TOPK = 100
```

Уменьши первые две переменные для быстрого пробного запуска.

## Результаты

В конце скрипт выводит Recall@20 и NDCG@20 для исходного порядка EASE и после
реранкинга. Также сохраняются:

- `/kaggle/working/capstone_metrics.json`;
- `/kaggle/working/rerank_lgbm.txt`.

Первый файл содержит метрики и важность признаков, второй - обученную модель LightGBM.
